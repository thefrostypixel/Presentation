Was ist JavaScript und

warum ist es so beliebt?

JavaScript, oft als JS abgekürzt,

ermöglicht Seiten im Internet interaktivität

98,8% aller Internetseiten nutzen es

Mosaic war 1993 der erste beliebte Browser,

hatte aber noch kein CSS oder JavaScript

Ein Titel

Darunter könnte ein Text stehen, der auf etwas verlinken kann, aber dekorierte Knöpfe waren nicht möglich.

Dekorierter Knopf

1994 erschien Netscape Navigator,

bekam 1995 Java Applets

und später JavaScript

JavaScript, anfangs LiveScript genannt,

wurde wegen Javas Beliebtheit umbenannt

1996 veröffentlichte Microsoft Internet Explorer

mit umbenanntem JavaScript namens JScript

er wurde schnell der beliebteste

Dekorierter Knopf

1997 wurde JavaScript von ECMA

als ECMAScript standardisiert

so kann JavaScript in jedem

Browser gleich funktionieren

ECMA entwickelte ECMAScript weiter

1998 erschien ECMAScript 2

mit vielen kleinen Verbesserungen

1999 erschien ECMAScript 3

mit try-catch Fehlerbehandlung

und Regulären Ausdrücken kurz RegEx,

welche das Suchen in Texten erleichtern

Mozilla veröffentlichte 2004 Firefox,

welcher 2005 ECMAScript unterstützte

Mozilla & Adobe arbeiteten an ECMAScript 4,

jedoch wurde es nie veröffentlicht

2008 erschien Google Chrome,

der durch die neue V8 Engine sehr schnell war

Googles neue Engine kompilierte

während ältere Engines interpretierten

um mitzuhalten, wechselten viele zu V8,
oder verbesserten ihre eigenen Engines

2022 wurde Internet Explorer eingestellt

und von Microsoft durch Edge ersetzt,

welcher ebenfalls auf V8 basiert

JavaScript ist Objekt Orientiert

Objekte haben Werte und Funktionen

{
  eineZahl:-123.45,
  einText:"Hallo",
  eineFunktion:function(){
    info("Welt");
  }
}

JavaScripts Ereignis Modell

andere Sprachen fragen ständig nach Ereignissen

in JavaScript kann einfach darauf gewartet werden

Funktionen können auch nach einer

bestimmten Zeit ausgeführt werden

oder in

einem Intervall ausgeführt werden

JavaScripts nutzt nur einen Thread,

weshalb mehrere Aufgaben nicht
gleichzeitig ausgeführt werden können

dieser Thread ist für alles zuständig

langsame Programme halten alles auf

JavaScripts hat keine Typisierung

falsche Wert-Typen könne ein Problem sein

TypeScript fügt Typisierung hinzu

var eineZahl:Number=123;

eineZahl="Hallo";

info(eineZahl-2);

Funktionen können wie Werte behandelt werden

sie können gespeichert oder weiter gegeben werden

var eineFunktion = function(){
  info("Hallo");
}

andereFunktion(eineFunktion);

Auf Leinwände kann gezeichnet werden

das ermöglicht zum Beispiel

Grafik-Rechner oder 3D-Spiele

Audios und Videos können abgespielt werden

und JavaScript ermöglicht es, sie zu steuern

audio.play();
video.pause();

IFrames erlauben, Webseiten einzubetten

zum Beispiel Karten oder Videos

Cookies können Informationen speichern

zum Beispiel Anmeldedaten

aber auch Unerwünschtes

Benutzername: Mein Name
Passwort: KeinGutesPasswort123!
IP Adresse: 12.345.678.90
Standort: Deutschland

JavaScript ermöglicht aber auch Werbung

indem es diese von einem Anbieter lädt

und in die Seite einbettet

JavaScript kann Elemente

finden, verändern und löschen

und neue Elemente erstellen

<div id="kasten">
  <p id="text">
    Hallo Welt!
  </p>
</div>
Was anderes

  <h1 id="neuer-titel">
    Ein Titel
  </h1>

So können Tabs gewechselt werden

indem einer versteckt und

der andere gezeigt wird

Oder Knöpfe funktionieren nur

wenn ein Kästchen angekreuzt ist

Klickbarer Knopf

JavaScript ermöglicht Anpassungsfähigkeit

zum Beispiel mit einem Themen-Knopf

Dynamisches Laden von Inhalten

ermöglicht Seiten wie TikTok

die unendlich Rollen können

Es gibt viele JavaScript-Engines

Firefox nutzt SpiderMonkey

Safari nutzt JavaScriptCore

Internet Explorer nutzte Chakra

alle anderen wie Opera, Brave

und Edge nutzen Chromes V8

Es gibt viele JavaScript-Engines

die alle gleich funktionieren sollen

IE hatte lange Zeit andere Funktionen

doch das hatte auch Gutes, da

Vorteile übernommen wurden

Heutzutage sind die einzigen Unterschiede

Geschwindigkeit und Arbeitsspeicher Verbrauch

Handys werden immer beliebter

rund die Hälfte aller Nutzer nutzen sie

kleine Bildschirme und Touchscreens

müssen daher auch unterstützt werden

Frameworks & Bibliotheken

Frameworks sind strukturierte Grundlagen

Bibliotheken enthalten nützliche Funktionen

Electron ermöglicht es, aus Webseiten

Desktop Anwendungen zu erstellen

diese funktionieren auf allen Betriebssystemen

und können zum Beispiel auf Dateien zugreifen

Allerdings ist Electron langsamer und ineffizienter

und jedes Programm enthält Google Chrome

Node.js ermöglicht es, JavaScript und

TypeScript auf einem Server auszuführen

es wird über die Konsole gesteuert

und nutzt Googles V8 Engine

Ein wichtiger Teil von Node.js ist npm

der Node Package Manager

Nutzer können Module hochladen

und Andere können diese nutzen

wodurch sie sich Arbeit sparen

Kürzlich wurde Bun veröffentlicht

welches eine Alternative zu Node.js ist

und Apples JavaScriptCore nutzt

Es enthält eine eigene, oft einfachere, API

und einen eigenen Package Manager

Eigene Meinung

JavaScript hat Internetseiten revolutioniert

ohne es wäre das Internet nicht so beliebt

Eigene Meinung

Electron Anwendungen finde ich nicht so gut

da sie eher langsam und unnötig groß sind

aber ich verstehe, warum Entwickler sie nutzen

Eigene Meinung

Es gibt viele nützliche Frameworks & Bibliotheken

aber die große Anzahl kann für Anfänger zu viel sein

Eigene Meinung

Insgesamt finde ich JavaScript gut geeignet

da es viel kann und einfach zu lernen ist und

Internetseiten oft nicht viel Leistung brauchen
