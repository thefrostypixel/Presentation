Was ist JavaScript und

warum ist es so beliebt?

JavaScript, oft als JS abgekürzt,

ermöglicht Seiten im Internet interaktivität

98,8% aller Internetseiten nutzen es

Mosaic war 1993 der erste Browser,

hatte aber noch kein JavaScript

Ein Titel

Ein Untertitel

Darunter steht dann ein Text, der sogar auf etwas verlinken, mehr war allerdings noch nicht möglich.

Dekorierter Knopf

1994 erschien Netscape Navigator,

bekam 1995 Java Applets

und später JavaScript

JavaScript, anfangs LiveScript genannt,

wurde wegen Javas Beliebtheit umbenannt

1996 veröffentlichte Microsoft Internet Explorer

mit CSS & JScript (umbenanntes JavaScript)

er wurde schnell der beliebteste

Dekorierter Knopf

1997 wurde JavaScript von ECMA

als ECMAScript standardisiert

so kann JavaScript in jedem

Browser gleich funktionieren

ECMA entwickelte es weiter

1998 erschien ECMAScript 2

mit dem Modul-System

und Try, Catch & Finally

1999 erschien ECMAScript 3

mit JavaScript Object Notation

und neuen Datentypen

Mozilla veröffentlichte 2004 Firefox

welcher 2005 ECMAScript unterstützte

Mozilla & Adobe arbeiteten an ECMAScript 4

jedoch wurde es nie veröffentlicht

2008 veröffentlichte Google Chrome

der durch die V8 Engine sehr schnell war

Googles neue Engine kompilierte

während ältere Engines interpretierten

um mitzuhalten, wechselten viele zu V8,
oder verbesserten ihre eigenen Engines

2020 wurde Internet Explorer eingestellt

und von Microsoft durch Edge ersetzt

welcher ebenfalls auf V8 basiert

JavaScript ist Objekt Orientiert

Objekte haben Werte und Funktionen

{
  eineZahl:-123.45,
  einText:"Hallo",
  eineFunktion:function(){
    info("Welt");
  }
}

JavaScripts Ereignis Modell

andere Sprachen fragen ständig nach Ereignissen

in JavaScript kann einfach darauf gewartet werden

Funktionen können auch nach einer

bestimmten Zeit ausgeführt werden

oder in

einem Intervall ausgeführt werden

JavaScripts nutzt nur einen Thread

deshalb können mehrere Aufgaben
nicht gleichzeitig machen

dieser Thread ist für alles zuständig

langsame Programme halten alles auf

JavaScripts hat keine Typisierung

falsche Wert-Typen könne ein Problem sein

TypeScript fügt Typisierung hinzu

var eineZahl:Number=123;

eineZahl="Hallo";

info(eineZahl-2);

Funktionen können wie Werte behandelt werden

sie können gespeichert oder weiter gegeben werden

var eineFunktion=function(){
  info("Hallo");
}

andereFunktion(eineFunktion);

audio.play();

IFrames erlauben, Webseiten einzubetten

zum Beispiel Karten oder Videos

Frameworks & Bibliotheken

Frameworks sind strukturierte Grundlagen

Bibliotheken enthalten nützliche Funktionen

Bibliotheken

Frameworks

React

- Ist das beliebteste Framework
- Nutzt ein modulares System
- Wird von Facebook entwickelt
- Der Quelltext ist frei verfügbar
- Nutzt JSX für Benutzeroberflächen
- Hat ein Ökosystem mit vielen
   Bibliotheken und Werkzeugen

Angular

- Wird von Google entwickelt
- Hat viele integrierte Funktionen und
   hilfreiche Werkzeuge
- Ist für Anwendungen mit nur einer Seite
- Nutzt hauptsächlich TypeScript
- Nutzt Dekoratoren und Annotationen für
   die Konfiguration von Komponenten

Vue.js

- Sehr kompakt und leichtgewichtig
- Ist sehr leicht zu lernen
- Kann schrittweise integriert werden
- Benutzt ebenfalls Komponente
- Der Server kann leicht mit dem
   Terminal bedient werden

Svelte

- Wird kompiliert, wodurch es schneller
   und leichtgewichtiger ist
- Erleichter das Erstellen von Animationen
- Hat einen sehr kompakten Syntax
- Vereinfacht das Teilen von Quelltext
   in kleinere Module

Next.js & Nuxt.js

- Next.js erleichtert die Nutzung von React,
   Nuxt.js hingegen vereinfacht Vue.js
- Können die Seite als eine Datei liefern
- Sind durch Erweiterungen anpassbar
- Komprimieren den JavaScript Quelltext
- Ermöglichen komplexe URLs, die unter
   anderem Variablen enthalten können

jQuery

- Vereinfacht DOM-Manipulation
   und Ereignis-Verarbeitung
- Reduziert Schreibaufwand und Quelltext
- Unterstützt CSS-Selektoren
- Ermöglicht asynchrone Kommunikation
   mit dem Server
- Vielen Frameworks enthalten Ähnliches

D3.js

- Ermöglicht Visualisierung von Daten
- Kann Elemente mit Daten verknüpfen
- Kann die Daten als SVGs darstellen
- Enthält hilfreiche Werkzeuge für
   das Arbeiten mit Daten
- Sehr kompakt und leichtgewichtig
- Der Quelltext ist frei verfügbar

Lodash

- Erleichtert die Arbeit mit Datenstrukturen
   wie Listen und Objekte
- Sehr schnell und effizient
- Besteht aus Modulen, die einzeln
   genutzt werden können
- Mehrere Funktionen für Objekte können
   aneinander gereiht werden

Three.js

- Ermöglicht die Darstellung von
   3D Objekten im Webbrowser
- Kann Modelle aus einer Datei Laden
- Erlaubt das Abspielen von Animationen
- Unterstützt komplexe Texturen
- Nutzt WebGL und damit die deutlich
   schnellere Grafikkarte

Electron ermöglicht es, aus Webseiten

Desktop Anwendungen zu erstellen

diese funktionieren auf allen Betriebssystemen

und können zum Beispiel auf Dateien zugreifen

Allerdings ist Electron langsamer und ineffizienter

und jedes Programm enthält Google Chrome

Node.js ermöglicht es, JavaScript und

TypeScript auf einem Server auszuführen

es wird über die Konsole gesteuert

und nutzt Googles V8 Engine

Ein wichtiger Teil von Node.js ist npm

der Node Package Manager

Nutzer können Module hochladen

und Andere können diese nutzen

wodurch sie sich Arbeit sparen

Kürzlich wurde Bun veröffentlicht

welches eine Alternative zu Node.js ist

und Apples JavaScriptCore nutzt

Es enthält eine eigene, oft einfachere, API

und einen eigenen Package Manager

JavaScript wurde 1995 entwickelt

um Webseiten interaktiv zu gestalten

es kann den Inhalt von Seiten ändern

und auf Ereignisse reagieren

es kann für reguläre Anwendungen

und Server genutzt werden

es gibt viele Frameworks & Bibliotheken

die das Entwickeln vereinfachen

aber es hat auch viele Nachteile

wie dass es viel Arbeitsspeicher braucht

und keine Typisierung hat

trotzdem war JavaScript in den

letzten Jahren die beliebteste Sprache
